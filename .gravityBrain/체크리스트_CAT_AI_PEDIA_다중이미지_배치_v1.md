# 냥(Nyan) - 다중 이미지 렌더링 및 UI 배치 구현 계획서 (v1.0)

## 📌 목표
- 각 품종별 다양한 각도(전신, 정면, 귀여운 표정)와 대표 색상(3가지)을 보여줄 수 있는 초고화질/초실사(Photorealistic) 이미지 렌더링 파이프라인 확장.
- 생성된 다양한 이미지들을 '박물관' 콘셉트를 해치지 않는 선에서 미니멀하고 고급스럽게 화면에 배치(UI/UX).

## 🕵️‍♂️ MASC 분석 (다중 페르소나 검토)
- **PM (기획/UX)**: 사용자가 고양이의 매력을 다각도에서 느낄 수 있어 리텐션과 체류 시간이 폭증할 것입니다. 다만 렌더링 시간이 길어지므로, 우선 핵심 20종에 대해서만 필수 각도와 색상을 정의하여 효율성을 높여야 합니다.
- **Architect (설계/기술)**: `BreedData` 스키마에 새로운 이미지 필드(`image_frontal`, `color_variants` 등)를 추가해야 합니다. 또한 `flux_gen.py`가 동적 프롬프트(예: 색상 변수 주입)를 지원하도록 구조를 개선해야 합니다. UI는 Framer Motion을 활용한 횡스크롤 갤러리나 모달 형태가 적합합니다.
- **Security (보안/안정성)**: 로컬 RTX 5080의 VRAM 부하와 스토리지 용량을 고려해야 합니다. 한 번에 너무 많은 렌더링을 시도하지 않도록 스크립트에 Batch 제한과 중단/재개 로직이 반드시 필요합니다.
