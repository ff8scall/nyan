# 🕵️ 품종 시각 검수 프로토콜 (Breed Visual Audit Protocol)

전문가가 아니어도 400+장의 에셋 중 '실패작'을 효율적으로 골라내기 위한 가이드라인입니다.

## 1. 1단계: AI 비전 교차 검증 (자동화 권장)
사용자가 직접 눈으로 확인하기 전, AI 비전 모델을 감별사로 활용합니다.
- **방법**: 생성된 `*.png` 파일을 GPT-4o나 Gemini 1.5 Pro 같은 멀티모달 AI에게 전달.
- **질문 프롬프트**: 
  > "이 사진은 [품종명] 고양이로 생성된 이미지야. 이 품종의 표준 특징(예: [JSON 내 특징])과 비교했을 때, 이 사진 속 고양이가 해당 품종이 맞는지 1~10점으로 평가해줘. 특히 [킬러 트레이트]가 누락되었다면 0점을 줘."
- **결과**: 5점 이하인 이미지들만 리스트업하여 사용자에게 보고.

## 2. 2단계: 킬러 트레이트(Killer Traits) 집중 타격
모든 디테일을 보지 말고, 해당 품종의 **'정체성'**만 확인합니다. 아래는 주요 체크포인트 예시입니다.

| 품종 그룹 | 킬러 트레이트 (이것만 보세요) | 실패 시 증상 |
| :--- | :--- | :--- |
| **체형 특화** | 먼치킨(다리 길이), 맹크스/킴릭(꼬리 유무) | 다리가 길거나 꼬리가 길면 무조건 탈락 |
| **귀 특화** | 아메리칸 컬(뒤로 휨), 스코티시 폴드(앞으로 접힘) | 귀가 쫑긋 서 있으면 일반 고양이임 |
| **털 질감** | 라펌/셀커크 렉스(곱슬), 스핑크스(털 없음) | 직모거나 털이 복슬복슬하면 탈락 |
| **포인트 컬러** | 샴/버만/히말라얀 (얼굴, 발 끝만 어두움) | 온몸이 어둡거나 무늬가 섞이면 탈락 |
| **특이 외모** | 하바나 브라운(수염까지 갈색), 이집트 마우(스포티드) | 수염이 흰색이거나 점박이가 없으면 탈락 |

## 3. 3단계: 시각적 대조 도구 (Visual Comparison Tool)
브라우저에서 직접 확인하고 싶을 때 사용하는 방법입니다.
- **검수용 대조 페이지**: `.../ko/breeds/audit` 같은 숨겨진 페이지를 만들어 왼쪽엔 **AI 이미지**, 오른쪽엔 **구글 검색 결과 이미지**를 띄우는 인터페이스 구축.
- **색상 팔레트 체크**: 배경이 완전한 블랙(#000)이 아니거나 피사체에 다른 색상 노이즈가 낀 경우 즉시 재생성 대상.

## 4. 4단계: '유사성' 문제 해결 로직
"고양이 얼굴이 다 똑같아 보여요"라는 문제 해결 방법:
- **각도 다양화**: Master는 정면, Fullbody는 옆면, Macro는 눈동자 위주로 검수하여 '각도'로 차별성을 부여했는지 확인.
- **포즈 차별화**: 활동적인 품종(아비시니안)은 서 있는 자세, 얌전한 품종(렉돌)은 앉아있는 자세로 프롬프트가 적용되었는지 확인.

---

### 💡 사용자님을 위한 Quick Tip
지금 가장 걱정되시는 **'비슷비슷한 얼굴들'**은 사실 AI가 '예쁜 고양이' 한 가지만 그릴 때 발생합니다. 이를 방지하기 위해 제가 각 JSON에 **품종별 골격(Body Type: Cobby, Foreign, Oriental 등)** 데이터를 넣어두었습니다. 

만약 **"이놈이나 저놈이나 똑같다"** 싶으시면, 제가 나중에 **'골격을 더 과장되게 그리는 프롬프트'**로 일괄 교정해 드릴 수 있으니 걱정 마세요!
