# [Project: CAT-AI-PEDIA] Master Plan & Gap Analysis

## 1. 기존 기획(Bcat) vs 신규 기획(CAT-AI-PEDIA) 비교 분석

기존 `.gravityBrain`에 저장된 문서들(`PRODUCT_REQUIREMENTS.md`, `IMAGE_GENERATION_PLAN.md`, `ROADMAP.md` 등)과 새롭게 제시된 **'CAT-AI-PEDIA'** 프로젝트 정의서를 비교 분석한 결과입니다.

| 구분 | 기존 기획 (Bcat Guide) | 신규 기획 (CAT-AI-PEDIA) | 핵심 변화 (Pivot) |
| :--- | :--- | :--- | :--- |
| **핵심 가치** | 입양 전 정보 제공, SEO 기반 객관적 비교 | 시각적 완벽함, 초현실적 아트, 3D 인터랙션 | 정보 전달 백과사전에서 **'하이엔드 디지털 아트/도감'**으로 완전한 방향 전환 |
| **이미지/비주얼** | 사실 왜곡 방지용 참고 이미지 (경고 문구 포함) | 8K급 질감, Cinematic Lighting, 3D 변환 | 실사를 대체하는 수준을 넘어 **'사진보다 더 아름다운 마스터피스'** 생성 |
| **기술 스택** | Next.js (SSR 중심), 기본 SEO 최적화 | **RTX 5080 + 9800X3D**, Flux.1, 3D(Meshy/Splatting) | 고성능 로컬 하드웨어를 한계까지 활용하는 **AI/3D 렌더링 파이프라인** 도입 |
| **수익/산출물** | 트래픽 기반 광고(Ad), 제휴 마케팅(Affiliate) | Next.js 3D 갤러리 + **프리미엄 실물 도감(Artbook)** | 프리미엄 가치 창출을 통한 **실물 도서 판매(예약/크라우드펀딩)** 및 고가치 브랜딩 |
| **페르소나/AI역할** | 객관적인 동물학자, 정보 큐레이터 | **고양이 유전학 전문가이자 하이엔드 사진작가** | 사실성과 예술성을 동시에 만족시키는 **정밀한 프롬프트 엔지니어링** 필요 |

## 2. CAT-AI-PEDIA 상세 구현 로드맵 및 기술 계획

신규 비전을 달성하기 위해 하드웨어 자원과 AI 기술을 극대화하는 구체적 실행 계획입니다.

### Phase 1: 'Museum + Encyclopedia' 정보 아키텍처 및 SEO 데이터 구축 (Current Focus)
- **목표**: 이미지가 없어도 압도적인 깊이를 자랑하는 박물관급 텍스트/데이터베이스 완비.
- **인기순 카테고리화**: 트래픽/검색량이 가장 높은 40종의 고양이를 인기순/특징별(예: 체형, 털길이, 성격)로 그룹화하여 다차원 카테고리 트리 구축.
- **초정밀 데이터 스키마 (SEO 최적화)**: 
  - 단순 특징을 넘어 '역사적 기원(History)', '유전학 및 수의학적 팩트(Genomics & Health)', '오해와 진실(Myth vs Fact)', '행동 심리학(Behavioral Psychology)' 등 위키피디아를 뛰어넘는 깊이의 정보 구성.
- **실행**: `ENCYCLOPEDIA_STRUCTURE.md` 확립 및 40종 데이터베이스 작성.

### Phase 2: Next.js 럭셔리 갤러리 플랫폼 & SEO 배포 (Web MVP)
- **목표**: 텍스트 데이터만으로도 미려하게 작동하는 정적 사이트(SSG) 구축 및 검색 엔진 장악.
- **구현**: Next.js App Router, 다크 모드 기반의 프리미엄 UI, 타이포그래피 중심의 레이아웃.
- **SEO 훅(Hook)**: 스키마 마크업(FAQ, Article, AnimalBreed)을 적용하고, "메인쿤 유전병", "렉돌 털빠짐 진실" 등 롱테일 키워드(Long-tail keyword)를 타겟팅한 자동화된 콘텐츠 페이지 배포.

### Phase 3: 고해상도 2D 마스터피스 이미지 생성 (Deferred)
- **목표**: 준비된 정보 아키텍처 위에 화룡점정이 될 8K급 아트 이미지 입히기.
- **구현**: RTX 5080 + Flux.1 환경에서 품종별 생물학적 체크리스트를 반영한 초현실적 2D 렌더링 파이프라인 가동 (기존 Phase 1에서 후순위로 변경).

### Phase 4: 3D 인터랙티브 경험 확장 (Far Future)
- **목표**: 2D 이미지를 3D/Gaussian Splatting으로 변환. (현재 시점에서는 고려하지 않음)

## 3. 작업 착수 체크리스트 (Final To-Do List)

> **핵심 원칙**: 완벽한 한글+영문(i18n) 다국어 텍스트 구조와 Next.js 렌더링이 선행되어야 하며, 이미지 생성은 이 모든 뼈대가 완성된 후 최후방에서 진행합니다.

### Step 1: 다국어(한/영) 백과사전 데이터베이스 구축
- [ ] **Top 40 품종 리스트 확정**: 글로벌 및 한국 검색량을 교차 분석하여 40종 최종 선정.
- [ ] **다국어 스키마(JSON) 템플릿 제작**: 영문(`en`)과 국문(`ko`) 데이터를 1:1 매핑하는 `[slug].json` 구조체 확립.
- [ ] **LLM 기반 데이터 양산 파이프라인**: 6단계 딥 큐레이션(역사, 유전학, 심리학 등) 텍스트를 한/영 양면으로 자동 생성하는 스크립트 작성.

### Step 2: Next.js + 다국어(i18n) 라우팅 및 UI/UX 개발
- [ ] **Next.js App Router 초기 셋팅**: `/[lang]/breeds/[slug]` 형태의 다국어 라우팅 구조 구현.
- [ ] **매거진 레이아웃 컴포넌트 개발**: 다크 모드 기반, 스크롤 인터랙션(Sticky Nav + Fade-in)이 적용된 프리미엄 텍스트 뷰어 컴포넌트 제작.
- [ ] **다차원 카테고리/필터 UI**: '자이언트 그룹', '아파트 적응형' 등 인기순/특징별 필터링 기능 구현.

### Step 3: SEO 및 메타데이터 최적화
- [ ] **JSON-LD 동적 생성기**: 각 품종 페이지마다 다국어 `AnimalBreed`, `FAQPage` 스키마 마크업 자동 주입.
- [ ] **Hreflang 태그 적용**: 구글 봇이 한국어/영어 페이지를 정확히 구분하여 인덱싱하도록 `<link rel="alternate" hreflang="...">` 설정.

### Step 4: [FINAL] 고해상도 2D 이미지 생성 및 적용
- [ ] **AI 환경 셋팅**: RTX 5080 로컬 ComfyUI / Flux.1 셋팅.
- [ ] **마스터피스 렌더링**: 완성된 웹 구조의 '빈 액자(Placeholder)'를 채울 생물학적 고증 기반의 8K 2D 이미지 생성 및 QA.
