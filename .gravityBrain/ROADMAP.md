# Roadmap

## 1. Roadmap Philosophy

The project should be built in layers:
1. Strategy and data foundation.
2. Bilingual content structure.
3. Visual/image pipeline.
4. SEO launch.
5. Comparison and quiz features.
6. Monetization and expansion.

Do not start with complex features before the content model and page template are stable.

## 2. Phase 0: Foundation

### Goals
- Finalize brand direction.
- Confirm initial 30 breeds.
- Define data schema.
- Define editorial standards.
- Define image policy.

### Deliverables
- Product requirements.
- Information architecture.
- Content specification.
- Breed schema.
- Image generation plan.
- SEO strategy.

### Exit Criteria
- All planning documents complete.
- Initial breed list approved.
- Required page list approved.
- Image labeling policy approved.

## 3. Phase 1: Content System

### Goals
- Prepare the content data structure before site implementation.

### Deliverables
- 30 breed data records.
- English content drafts.
- Korean content drafts.
- FAQ sets.
- Similar breed mappings.
- Comparison target mappings.

### Exit Criteria
- Every breed has required structured fields.
- Every breed has at least 5 FAQs.
- Every breed has SEO title/description.
- Every breed has similar breed links.

## 4. Phase 2: Site MVP Build

### Goals
- Build indexable bilingual site.

### Deliverables
- Home page.
- Breed index.
- Breed detail template.
- English/Korean routing.
- Language switcher.
- Metadata.
- Sitemap.
- Basic responsive layout.

### Exit Criteria
- All 60 breed pages render.
- Metadata exists on all pages.
- Hreflang exists on all equivalent pages.
- No placeholder content.
- Mobile layout works.

## 5. Phase 3: Image MVP

### Goals
- Add realistic, reviewed images.

### Deliverables
- Image generation workflow.
- First test batch of 5 breeds.
- Image review checklist.
- Final image folder structure.
- Hero images for all 30 breeds.
- Alt text and source metadata.

### Exit Criteria
- Every breed has at least one approved image.
- AI-generated images are labeled.
- Images are compressed and web-ready.
- Rejected images are not published.

## 6. Phase 4: Comparison and Quiz

### Goals
- Add decision-support functionality.

### Deliverables
- Breed filter UI.
- Comparison selector.
- Comparison table.
- Quiz-lite questions.
- Quiz result logic.
- Result explanation copy.

### Exit Criteria
- User can compare 2-4 breeds.
- User can complete quiz and receive recommendations.
- Recommendations include reasons.
- Quiz results link to breed pages.

## 7. Phase 5: SEO Expansion

### Goals
- Expand beyond core breed pages.

### Deliverables
- Beginner-friendly breeds guide.
- Apartment-friendly breeds guide.
- Low-shedding breeds guide.
- Affectionate breeds guide.
- Large cat breeds guide.
- Long-haired cat care guide.
- Comparison landing pages.

### Exit Criteria
- Internal links connect guide pages to breed pages.
- Guide pages have metadata.
- Search Console configured after deployment.

## 8. Phase 6: Monetization

### Goals
- Add revenue without harming trust.

### Deliverables
- Affiliate disclosure.
- Editorial policy.
- Product recommendation blocks.
- Breed-specific starter kits.
- Email capture.

### Exit Criteria
- Affiliate links are disclosed.
- Product recommendations are relevant.
- No medical product claims.
- No intrusive monetization above core content.

## 9. Suggested Timeline

### Week 1
- Finalize brand, schema, and content templates.
- Start breed data records.

### Week 2-3
- Draft 30 English breed pages.
- Draft 30 Korean breed pages.

### Week 4
- Build or prepare site structure.
- Implement route and page template.

### Week 5
- Generate/review first image batch.
- Add image metadata and optimization.

### Week 6
- Add comparison and filter basics.
- Add quiz-lite.

### Week 7
- SEO QA.
- Content QA.
- Launch preparation.

### Week 8+
- Publish guide articles.
- Improve images.
- Add monetization carefully.

## 10. Major Dependencies

- Final breed list.
- Content source quality.
- Image generation feasibility.
- Choice of framework.
- SEO implementation quality.
- Review time for Korean localization.

## 11. Risk-Based Priorities

### Highest Priority
- Structured data.
- Page template.
- SEO metadata.
- Content quality.
- Image policy.

### Medium Priority
- Comparison UI.
- Quiz.
- Guide articles.

### Later Priority
- Monetization.
- Directory.
- User accounts.
- Community features.
